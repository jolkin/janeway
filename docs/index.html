Janeway

An HTTP-served execution engine for AI plans — RMPL, PDDL, and raw state plans, with built-in fault injection and a ROS 2 bridge.

Get started Read the docs

What is Janeway?

Janeway is a Docker-packaged service that takes an AI plan in any of three input formats — RMPL (the native Kirk format), PDDL (a domain, problem, and temporal plan), or a raw state plan JSON — and runs it end-to-end through planning, dispatch, and causal-link monitoring. Once the container is up, the entire surface is reachable over HTTP at http://localhost:8000.

What is it for?

Janeway is built for researchers and engineers who want a single, reproducible deployment of the Kirk planner and the PyKirk dispatcher, without rebuilding the pieces locally. It is well-suited to:

  • Submitting plans from a notebook or CI job and streaming back the resulting dispatch events.
  • Running fault-injection experiments against a known plan, either through a blanket-negation oracle or a YAML-driven spec.
  • Driving a real robot through the bundled ROS 2 bridge, with Janeway producing dispatch events and consuming execution reports.
  • Visually inspecting the planned and live execution graph in the browser through the included plan-visualization frontend.

What's inside?

Kirk

The Common Lisp planning engine. Takes an RMPL program or a state plan and produces a scheduled state plan.

PyKirk

Python dispatcher that drives a scheduled state plan through a local agent and oracle.

pddl_to_sp

Converts a PDDL domain, problem, and temporal plan into the state-plan JSON the planner expects.

Robust Execution

Causal-link monitor that tracks link integrity during dispatch and raises violations in real time.
